# Support

## Choose The Right Support Path

| Track | Where to ask | Boundary |
| --- | --- | --- |
| Personal Key / Community Client | GitHub issues for public bugs, build problems, docs gaps, and feature requests. | Do not paste keys, QR payloads, subscription URLs, personal connection links, account data, or unredacted logs. |
| Operator / Company Client | GitHub issues for public source contracts and fixture behavior. Operator-specific users should contact the operator that shipped their build. | Operators own their backend, signing, support, release channels, privacy policy, and user claims. |
| Official POKROV Service Mode | Official POKROV support links below. | Forks and operator builds are not official POKROV builds and are not supported as official POKROV apps. |

## Official POKROV Service Help

For official POKROV account, service, payment, or official app help, use:

- Support bot: https://t.me/pokrov_supportbot
- Website: https://pokrov.space/

## GitHub Issue Scope

GitHub issues are best for:

- Personal Key / Community Client bugs that can be reproduced without private
  keys or private subscription URLs
- Operator / Company Client fixture, OpenAPI, build define, and white-label
  documentation gaps
- build-from-source problems
- public documentation corrections
- feature requests that keep the public boundary honest

GitHub issues are not for:

- official POKROV account, payment, or service support
- private operator customer support
- secrets, tokens, QR payloads, subscription URLs, personal connection links,
  signing material, or private backend details
- vulnerability reports

## Security Support

Do not open public issues for vulnerabilities, exploit details, private account
problems, private operator infrastructure, or abuse instructions. Read
[SECURITY.md](SECURITY.md) first.

## Fork Support

Forks are responsible for their own builds, backend compatibility, release
claims, and user support. The official POKROV support channels cannot guarantee
support for modified clients.
